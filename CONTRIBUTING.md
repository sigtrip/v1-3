# Contributing to Argos Universal OS

Спасибо, что хотите помочь проекту Argos.

## Как начать

1. Форкните репозиторий и создайте ветку от `main`.
2. Установите зависимости:
   - `pip install -r requirements.txt`
   - (опционально для расширенных модулей) `pip install -r requirements-optional.txt`
3. Инициализируйте проект:
   - `python genesis.py`
   - Если используете `.env`, его можно держать как в текущей рабочей директории запуска, так и в корне репозитория (используется bootstrap с fallback).
4. Проверьте целостность окружения:
   - `python health_check.py`
5. Запустите тесты:
   - `pytest -q`

## Что можно улучшать

- Новые `skills` в `src/skills/`
- Интеграции IoT и bridge-модули в `src/connectivity/`
- Улучшение наблюдаемости (`src/observability.py`)
- Улучшение TUI/UX и сценариев в `examples/`
- Документация в `docs/`

## Стандарты кода

- Python 3.10+
- Малые, сфокусированные PR
- Без хардкода секретов
- Не добавлять лишние зависимости без необходимости
- Для optional-зависимостей (SDR, Watsonx, BLE и т.д.) использовать graceful fallback, чтобы базовый запуск не ломался
- Для новой функциональности добавлять тесты в `tests/` (если есть покрытие рядом)

## Процесс Pull Request

1. Обновите код и документацию
2. Убедитесь, что проверки проходят:
   - `python health_check.py`
   - `pytest -q`
3. Опишите:
   - что изменено,
   - почему,
   - как проверяли
4. Если меняется поведение CLI/TUI — приложите пример команды и вывода

## Security

Не публикуйте API-ключи, токены и приватные данные.
Для уязвимостей используйте приватный disclosure через владельца репозитория.
